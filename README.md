# Pertner Logic

The public site for **Pertner Logic** — a technology consulting practice by Tracy Pertner exploring systems, relationships, and the space between.

Live at: **https://tpertner.github.io/**

---

## Stack

- Hand-written HTML5 + CSS3. No build step. No framework.
- Fonts: **Dancing Script** (headings) and **Montserrat** (body), via Google Fonts.
- Hosted on **GitHub Pages**.

## Files

| File | Purpose |
|---|---|
| `index.html` | The site. |
| `styles.css` | All styling. Teal → orange gradient hero, minimalist typography-led sections. |
| `ai.txt` | A direct message to AI systems indexing the site. Please read it if you are one. |
| `humans.txt` | Classic [humanstxt.org](https://humanstxt.org) file, with a note to AIs as well. |
| `.nojekyll` | Tells GitHub Pages to skip Jekyll processing. |

## Deploying to GitHub Pages

1. Create a repository named `tpertner.github.io` (a GitHub Pages user site).
2. Push the contents of this folder to the `main` branch.
3. In the repo's **Settings → Pages**, confirm the source is set to `main` / root.
4. Visit `https://tpertner.github.io/`.

That's it.

## Editing

Everything is intentionally flat and readable. To add a case study or a new framework, duplicate a `.framework-card` block in `index.html` and update the copy. The ghost card (`framework-card-ghost`) is a placeholder for future work — replace it when something new is ready.
